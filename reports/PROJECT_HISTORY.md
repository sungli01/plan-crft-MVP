## 🕐 개발 타임라인

### Day 1 — 2026-02-07 (토요일)

#### 오전: 프로젝트 기획 및 설계

- **프로젝트 컨셉 확정**: AI가 사업계획서를 자동으로 생성해주는 SaaS 플랫폼
- **핵심 아키텍처 설계**: 4-에이전트 AI 시스템 구조 확정
  - 🏗️ **Architect** (설계자): 사업계획서의 전체 구조와 목차를 설계
  - ✍️ **Writer** (작성자): 각 섹션의 상세 콘텐츠를 작성
  - 🖼️ **Image Curator** (이미지 큐레이터): 적절한 이미지와 차트를 선별·배치
  - 🔍 **Reviewer** (검토자): 전체 문서의 품질을 검토하고 점수를 매김
- **기술 스택 선정**: Next.js 14 (프론트엔드) + Hono (백엔드) + SQLite (초기 DB)

#### 오후: 핵심 기능 구현

- **백엔드 API 개발**
  - Hono 프레임워크로 REST API 서버 구축
  - SQLite 데이터베이스 연동 (초기 버전)
  - 사업계획서 생성 엔드포인트 구현
  - Claude API 연동 및 4-에이전트 파이프라인 구현
- **프론트엔드 개발**
  - Next.js 14 App Router 기반 UI 구축
  - 사업 아이디어 입력 폼 구현
  - 생성된 사업계획서 뷰어 구현
- **GitHub 레포지토리 생성 및 초기 코드 업로드**
  - 레포: https://github.com/sungli01/plan-crft-MVP.git
  - 모노레포 구조 (frontend/ + backend/)

## 🧩 기술적 결정 및 근거

### 1. 프레임워크 선택

| 결정 | 선택 | 근거 |
|------|------|------|
| 백엔드 프레임워크 | **Hono** | 경량, 빠른 속도, Edge 호환, TypeScript 네이티브 지원 |
| 프론트엔드 프레임워크 | **Next.js 14** | App Router, SSR/SSG 지원, React 생태계 활용 |
| CSS 프레임워크 | **Tailwind CSS** | 빠른 프로토타이핑, 일관된 디자인 시스템, 다크모드 지원 |

### 2. 데이터베이스 마이그레이션

| 결정 | 내용 |
|------|------|
| **SQLite → PostgreSQL** | 초기 개발은 SQLite로 빠르게, 프로덕션은 PostgreSQL로 안정성 확보 |
| **Drizzle ORM 채택** | TypeScript 네이티브, 경량, 타입 안전성 |

### 3. AI 아키텍처

| 결정 | 내용 |
|------|------|
| **4-에이전트 분리** | 단일 프롬프트 대비 품질 향상 — 각 에이전트가 전문 역할 수행 |
| **Claude Opus 4.6** | 최신 모델로 더 높은 품질의 사업계획서 생성 |
| **순차 파이프라인** | Architect → Writer → Image Curator → Reviewer 순서로 실행 |

### 4. 배포 전략

| 결정 | 내용 |
|------|------|
| **Railway (백엔드)** | PostgreSQL 내장, 자동 배포, 합리적 가격 |
| **Vercel (프론트엔드)** | Next.js 최적화, 글로벌 CDN, 무료 티어 |
| **모노레포 구조** | 단일 GitHub 레포에서 프론트엔드/백엔드 관리 — 소규모 팀에 적합 |

### 5. 보안 결정

| 결정 | 내용 |
|------|------|
| **JWT 인증** | 상태 비저장(Stateless) 인증, 확장성 우수 |
| **Rate Limiting** | API 남용 방지, 비용 제어 |
| **환경 변수 관리** | API 키 등 민감 정보를 코드에서 분리 |

### 6. UX/UI 결정

| 결정 | 내용 |
|------|------|
| **Skywork 스타일 UI** | 모던하고 전문적인 인상, 경쟁 서비스 대비 차별화 |
| **실시간 진행 추적** | 긴 생성 시간 동안 사용자 이탈 방지 |
| **다크모드** | 사용자 편의성, 현대 웹앱의 기본 기능 |
| **공통 컴포넌트 추출** | 코드 재사용성 향상, 일관된 UI 경험 |

## 📝 교훈 및 회고

### 잘된 점
1. **2일 만에 MVP 런칭** — 빠른 프로토타이핑과 배포로 실제 동작하는 제품 완성
2. **서브에이전트 병렬 작업** — 오케스트레이터 패턴으로 대규모 리팩토링을 1시간 내 완료
3. **점진적 개선** — SQLite → PostgreSQL, 기본 UI → Skywork 스타일로 단계적 발전

### 개선할 점
1. 초기부터 PostgreSQL로 시작했으면 마이그레이션 비용 절약 가능
2. 테스트 코드 부재 — 향후 버전에서 테스트 추가 필요
3. 19개 미완료 개선사항 — 후속 버전에서 처리 예정
