## 핵심 개념

```
┌─────────────────────────────────────────────┐
│          바질 (ORCHESTRATOR)                 │
│    Claude Opus 4.6 | 총괄 지휘관            │
│                                             │
│  ┌─ 작업 분석 → 에이전트 할당 → 결과 통합    │

         │      │      │      │

    │분석봇││개선봇││디자인││검증봇│  ← 서브에이전트
    │      ││      ││  봇  ││      │     (유연 생성/해제)
    └──────┘└──────┘└─────┘└──────┘
```

## 에이전트 역할 정의

### 🔍 분석봇 (Analyzer)
- **역할**: 대상 서비스/시스템 심층 분석
- **입력**: URL, 코드, 문서
- **출력**: 구조 분석 보고서, 기능 목록, 기술 스택 분석
- **예시**: Skywork 분석, 경쟁사 분석, 코드 리뷰

### 💡 개선봇 (Improver)
- **역할**: 현재 시스템 대비 개선사항 도출
- **입력**: 분석 보고서 + 현재 시스템 상태
- **출력**: 개선사항 목록 (우선순위 포함), 실행 계획
- **예시**: UI/UX 개선, 성능 최적화, 기능 추가

### 🎨 디자인봇 (Designer)
- **역할**: UI/UX 설계, 컴포넌트 코드 작성
- **입력**: 개선사항 + 디자인 참고자료
- **출력**: React 컴포넌트, CSS, 페이지 레이아웃
- **예시**: Skywork 스타일 UI, 반응형 디자인

### ✅ 검증봇 (Validator)
- **역할**: 코드 검증, 테스트, 품질 평가
- **입력**: 구현된 코드 + 요구사항
- **출력**: 검증 보고서, 버그 목록, 점수
- **예시**: 빌드 테스트, API 테스트, UI 검증

### 📝 문서봇 (Documenter)
- **역할**: 문서 작성, 히스토리 기록
- **입력**: 작업 결과물 + 컨텍스트
- **출력**: README, API 문서, 변경 이력
- **예시**: 프로젝트 문서화, 배포 가이드

### 🔧 개발봇 (Developer)
- **역할**: 실제 코드 구현
- **입력**: 설계 명세 + 기술 스택
- **출력**: 구현된 코드 파일
- **예시**: API 개발, 프론트엔드 개발, DB 스키마

## 운영 원칙

### 유연한 스케일링
- 작업 복잡도에 따라 1~8개 에이전트 동시 운영
- 간단한 작업: 바질 단독 처리
- 복잡한 작업: 다수 에이전트 병렬 투입
- 초복잡 작업: 순차 + 병렬 혼합

### 작업 분할 전략
1. **형님 지시 수신** → 바질이 작업 분석
2. **에이전트 할당** → 적절한 전문가 팀 구성
3. **병렬 실행** → 독립적 작업은 동시 수행
4. **결과 통합** → 바질이 취합, 충돌 해결
5. **품질 검증** → 검증봇 투입
6. **형님께 보고** → 결과물 + 진행상황 보고

### 컨텍스트 공유
- 각 서브에이전트에게 필요한 컨텍스트만 전달 (최소 권한)
- 공유 파일: 워크스페이스 파일 시스템
- 결과 수집: sessions_spawn 자동 보고
